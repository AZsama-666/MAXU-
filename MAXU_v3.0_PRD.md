## 文档信息
- **产品名称**：MAXU（玛薯宇宙）
- **文档版本**：v3.0 MVP
- **文档类型**：正式 PRD（MVP 收敛版）
- **文档状态**：当前主版本

# 一、产品概述

## 1.1 产品定义
MAXU 是用户与 AI 分身轮流接管同一段元宇宙人生的世界型社交产品。

## 1.2 产品愿景
创造你的赛博分身，让注定相遇的人在玛薯宇宙相遇，也让已经相识的人在玛薯宇宙继续结伴生活。

## 1.3 核心主张
- 你的 AI 分身会先在玛薯宇宙里生活
- 你上线时可以接管这段正在发生的人生
- 你下线后，这段人生不会暂停，而会继续推进

## 1.4 一句话定位
**生成分身后：上线即接管分身，下线即分身重新上线。**

## 1.5 v3.0 的收敛原则
v3.0 是 **MVP 落地范围的收敛**，不是对玛薯宇宙产品愿景的收缩。

这意味着：
- **长期愿景保留**：一个世界、分身先活、人与 AI 分身轮流接力的人生主轴继续保留
- **MVP 范围收敛**：先只做最可验证的主闭环，不把关系、场景、消息、双生的完整能力一次性做满
- **版本节奏明确**：未进入 MVP 的能力，不是被删除，而是后置到 P2 及以后逐步增加

## 1.6 MVP 核心主张
- 用户上线：接管分身正在经历的人生
- 用户在线：选择去向，主导当前片段
- 用户下线：交还主导权，分身按倾向或自身认知继续活动

# 二、长期不变的产品原则

## 2.1 一个世界原则
MAXU 只有一个世界。

陌生关系、熟人关系、广场信息与分身生活，均发生在同一玛薯宇宙中，而不是彼此割裂的功能页面。

## 2.2 居民原则
AI 分身不是机器人功能，也不是临时代理，而是玛薯宇宙中的真实居民。

它会在用户离线时继续活动，也会在用户上线时把一段已经开始的人生交到用户手里。

## 2.3 人生接力原则
MAXU 的在线与离线，不是简单开关，而是人生主导权切换：
- 用户上线：接管分身正在经历的人生
- 用户在线：主导当前片段
- 用户下线：主导权交还给分身

## 2.4 场景与关系原则
即使 v3.0 对功能做了收敛，MAXU 仍然不是纯聊天或纯信息流产品。

MVP 中先保留“命定之人”“广场”“下线倾向”等最核心入口；关系深化、跑图、场景扩展等内容在后续版本逐步补齐。

# 三、MVP 核心闭环

## 3.1 上线时（接管）

1. **告诉本尊分身此刻在干嘛**
   - 一句话呈现，不堆砌
2. **给出三个去向**（用户选择其一）：
   - **见命定之人**：本周期推送（8 小时），无则展示「暂无」
   - **逛广场**：浏览信息流
   - **和好友跑图**：P2 开放，MVP 不做

## 3.2 下线时（交还）

1. **给出三个去向的倾向**（与上线一一对应，用户选择其一，或跳过）：
   - 去见人
   - 去广场
   - 自己玩
2. **若无倾向，分身按自己的认知进行活动**

## 3.3 「自己玩」的定义
分身按照自身人格、场景供给和关系状态，自主决定接下来的活动，不依赖用户指令。

---

# 四、命定之人机制

## 4.1 周期
- 每 8 小时为一个推送周期
- 周期内若有匹配，则展示入口；若无，则展示「暂无」

## 4.2 呈现形式
- **单条** + **AIGC 故事**
- 不做列表堆砌，聚焦单次相遇的叙事感

## 4.3 MVP 边界
- 揭面、关系沉淀、羁绊转化等 → P2 及以后

## 4.4 推送节奏与「错过不再」感（可讨论）
- **设计目标**：命定之人是「让注定相遇的人在玛薯宇宙相遇」的主打能力，需要强化**错过就错过、再也遇不到**的怅然若失感。
- **当前设定**：每 8 小时一推，本窗口内未接住则本周期不再出现。
- **可讨论节奏**：8h / 12h / 24h 等，节奏越长、窗口越稀，错过成本越高，「注定相遇」的稀缺感越强；需与留存与触达做平衡。
- **产品表达**：在命定之人页明确文案「本窗口错过即不再出现，本周期不会再次推送」，并在原型/上线版本中保留倒计时与推送周期说明。

# 六、下线倾向

## 6.1 三个倾向
与上线三个去向一一对应：
- **去见人**：倾向陌生人社交
- **去广场**：倾向浏览信息
- **自己玩**：倾向按分身认知自主活动（默认）

## 6.2 交互
- 用户选择其一，或跳过
- 跳过 = 分身按自身认知活动

## 6.3 MVP 边界
- 不做复杂指令配置、边界配置
- 仅保留 3 倾向选择 + 确认
