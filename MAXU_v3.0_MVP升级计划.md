# 一、问题与动机

## 1.1 内部测试反馈

| 反馈 | 说明 |
|------|------|
| **文字过多** | 进入主页后内容密度高，用户体感负担大 |
| **玩法过重** | 设计过于复杂，MVP 阶段想全方面发展但处处一般 |
| **边界不清** | 需要明确 MVP 功能，其余内容通过后续版本逐步增加 |

## 1.2 升级动机

- 收敛到可验证的核心闭环
- 降低首屏认知负担
- 为后续版本迭代留出清晰扩展路径

# 二、MVP 定义（v3.0 核心）

## 2.1 一句话

**生成分身后：上线即接管分身，下线即分身重新上线。**

## 2.2 上线时（接管）

1. **告诉本尊分身此刻在干嘛**
   - 一句话呈现，不堆砌
2. **给出三个去向**（用户选择其一，文案简洁）：
   - **见命定之人**：本周期推送（8 小时），无则展示「暂无」；单条 + AIGC 故事
   - **逛广场**：浏览信息流
   - **和好友跑图**：P2 开放，MVP 不做

## 2.3 下线时（交还）

1. **给出三个去向的倾向**（与上线一一对应，用户选择其一，或跳过）：
   - 去见人 / 去广场 / 自己玩
2. **若无倾向，分身按自己的认知进行活动**

## 2.4 MVP 范围边界

| 功能 | MVP | P2 及以后 |
|------|-----|-----------|
| 命定之人（见命定之人） | ✓ 8 小时周期，单条+AIGC，无则「暂无」 | 揭面、关系沉淀等 |
| 广场（逛广场） | ✓ | 发布、互动深化 |
| 好友跑图 | ✗ | ✓ |
| 4 Tab（首页/羁绊/消息/双生） | ✓ 保留 | ✓ |
| 羁绊 / 消息 / 双生 内容 | 精简 | 完整功能 |
| 下线倾向 | ✓ 3 选 1 或跳过 | 指令、边界等 |
| Zone0 入世 | ✓ 完全保留 + 性格测试后 AI 自我对话（可跳过） | ✓ |

# 三、已敲定结论（Q1–Q8）

## 3.1 命定之人

- **Q1**：8 小时内无推送时 → **展示「暂无」状态**
- **Q2**：呈现形式 → **单条 + AIGC 故事**

## 3.2 下线 3 个倾向

- **Q3**：是否与上线 3 去向一一对应 → **是**
  - 倾向 A：去陌生人社交
  - 倾向 B：去广场
  - 倾向 C：自己玩（默认）
- **Q4**：「自己玩」的具体含义 → **按照分身自己的认知进行活动**

## 3.3 信息架构

- **Q5**：MVP 下 Tab 结构 → **需要，与目前 4 Tab 一致**（首页、羁绊、消息、双生）
- **Q6**：Zone0（入世）→ **完全保留**，同时增加：**做完性格测试题后与 AI 的自我对话过程，此流程可跳过**

## 3.4 文案与体感

- **Q7**：首屏「分身此刻在干嘛」→ **一句话即可**，目前版本文字太多
- **Q8**：三个去向的文案风格 → **建议简洁动词/短语**（见下方）

### Q8 文案建议（简洁）

| 去向 | 建议文案 | 备选 |
|------|----------|------|
| 陌生人社交 | 见命定之人 | 看看谁在等你 |
| 广场 | 逛广场 | 浏览世界动态 |
| 好友跑图 | 和好友跑图 | P2 开放 |

下线倾向同理，保持 2–4 字动词为主：**去见人** / **去广场** / **自己玩**。

## 4.2 文档结构建议（v3.0 PRD）

```
一、产品概述（精简）
二、MVP 核心闭环（上线接管 + 下线交还）
三、命定之人机制（8 小时周期）
四、广场信息流（MVP 范围）
五、下线倾向（3 选 1 或跳过）
六、暂不包含（P2 及以后）
七、版本记录
```

## 4.3 原型收敛建议

- 首屏：分身状态 + 3 去向，极简
- 命定之人：独立页或弹层，无则无入口
- 广场：单页信息流
- 下线：3 倾向选择 + 确认，无复杂指令配置
