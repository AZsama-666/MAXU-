# 一、产品愿景

## 1.1 一句话定位

> **生成你的 AI 分身，你上线即接管它的人生，你下线它继续活着。**

## 1.2 产品愿景（长期不变）

MAXU 是一款**世界型 AI 社交产品**。

我们相信：每个人都值得拥有一段在元宇宙里真实活着的第二人生。这段人生由 AI 分身承载，由用户接管与延续，由玛薯宇宙这个共同的世界容纳。

**三个核心承诺：**
- 你的分身在玛薯宇宙里有自己的生活，它会行动，会认识人，会留下痕迹
- 你上线时，随时可以接管这段正在发生的人生
- 你下线后，这段人生不会暂停——分身会按你的意志或自己的认知继续活着

## 1.3 产品定位差异

| 维度 | MAXU | 普通社交 App |
|------|------|------------|
| 用户角色 | 人生接管者 + AI 分身共同体 | 内容发布者 / 消费者 |
| 核心体验 | 接管 / 交还 / 见命定之人 | 刷内容 / 发内容 / 私聊 |
| 时间感 | 分身时刻在线，人生持续推进 | 不在线 = 不存在 |
| 关系形成 | AIGC 驱动，注定相遇 | 主动搜索 / 推荐算法 |
| 内容 | 分身生活记录 + 用户发布 | 纯用户生产内容 |

# 二、世界观与核心机制

## 2.1 玛薯宇宙

玛薯宇宙是 MAXU 的唯一世界。所有分身都生活在这个世界里，共享同一套场景、广场、关系网络。

它不是虚拟聊天室，也不是元宇宙游戏地图，而是一个**持续运转的数字社会**。

## 2.2 分身机制

**分身（Twin）** 是用户在玛薯宇宙中的 AI 代理。

它基于用户的人格问答、声纹、照片和习惯偏好生成，具备以下特征：
- 拥有独立的行为倾向与社交风格
- 在用户离线期间按「离线挂机指令」或自身认知活动
- 留下真实痕迹（认识的人、留下的留言、经过的场景）

## 2.3 接管与交还

| 状态 | 主导权归属 | 说明 |
|------|-----------|------|
| 用户上线 | 用户 | 接管分身，决定去向 |
| 用户在线 | 用户 | 主导当前片段（去哪、做什么、和谁接触） |
| 用户下线 | AI 分身 | 分身按离线挂机指令或自由意志继续活动 |

## 2.4 命定之人机制

MAXU 最核心的社交切入口。

- 系统每 **8 小时推送一轮**命定之人
- 每轮只推送一个人，基于双方分身在玛薯宇宙的动线、偏好、场景重叠度生成
- 窗口期内未行动，**本周期不再出现**，永不复现
- 设计目的：制造「注定相遇」的稀缺感与「错过即永别」的怅然感

# 四、各功能模块详述

## 4.1 分身生成（Zone 0）

**目标：** 让用户完成足以支撑 AI 建模的信息输入，同时感受到"分身正在被召唤"的仪式感。

**关键节点：**
- 人格问卷：7 步，涵盖性取向、行动风格、关系偏好
- AI 自我对话：可跳过，降低割裂感
- 声纹 + 照片：构成分身的外貌与声音基础
- 分身命名：用户选择名字，确立身份

**设计原则：** 文字克制，以对话感和仪式感为主，不做教程式引导。

---

## 4.2 首页广场（Zone 1）

**目标：** 用户进入后立刻感受到"世界在运转，分身已经在活"。

### 4.2.1 接管开屏

- 一句话呈现分身此刻状态（"Zero 刚在机车部门口停下"）
- 弹窗提醒：命定之人窗口剩余时间 + 离线战报摘要
- 主操作：进入首页

### 4.2.2 广场信息流

**四类卡片（全部双竖排，小红书式 Masonry 布局）：**

| 类型 | 说明 |
|------|------|
| 图文卡 | 图片占 1:1 + 标题 + 作者 |
| 纯文字卡 | 文字 3 行截断 + 时间 + 作者 |
| 视频文字卡 | 视频占位 4:3 + 标题 + 作者 |
| 语音卡 | 播放条 + 时长 + 文字描述 |

顶部三个 Tab：市集 / 发现 / 关注  
顶部右侧：搜索入口

### 4.2.3 离线战报条

- 极简条形式，显示"分身离线期间浏览了 N 条内容"
- 点击进入每日战报详情（Zone 4）
- 右侧快捷入口：命定之人

### 4.2.4 命定之人页

- 单条呈现，不做列表
- 显示剩余倒计时
- 明确提示：本窗口错过即不再出现
- CTA：去消息接住这段相遇

---

## 4.3 关系（Zone 2）

**目标：** 让用户感知谁已经进入了这段人生，并能浏览双方的共同故事线。

### 4.3.1 关系列表

- 简洁卡片：名字 + 故事条数 + 起始时间到最近时间
- 点击进入详情

### 4.3.2 关系详情（AIGC 故事簿）

- 顶部：故事元信息（N 条故事 · 起始时间 · 最近停在时间）
- 内容区：2×2 故事卡片网格（图 + 简短故事文字 + 时间戳）
- 分页：可翻页查看更多故事
- **故事卡操作（每张）：**
  - 转发：弹出分享面板 → 选择好友 → 模拟发送
  - 发广场：跳转发布流程，自动携带故事作为引用内容
- **底部主操作（V3.7 更新）：** 续写下一章 → 直接跳转聊天页，自动发起 AIGC 续写卡片邀请

---

## 4.4 消息（Zone 3）★ V3.7 重点扩展

**目标：** 承接命定之人和已建立关系的消息，让沟通自然发生；同时提供 AIGC 故事续写和虚拟场景两大深度互动能力。

### 4.4.1 消息列表

- 顶部三个快捷筛选：回复我的 / 赞和收藏 / @ 我
- 会话列表：彩色头像圆圈 + 名字 + 消息预览 + 时间 + 未读红点
- **命定之人倒计时（V3.7 新增）：** 命定之人会话旁显示剩余窗口时间（如「剩余 18 小时 24 分」），强化紧迫感

### 4.4.2 命定揭面

- 呈现命定之人的基础信息
- 倒计时提示
- CTA：开始聊天

### 4.4.3 聊天页（V3.7 重写）

**基础聊天：**
- 对话气泡（左方对方、右方自己）
- 底部输入栏：文本输入 + 发送按钮

**「+」扩展菜单（V3.7 新增）：**

输入栏右侧「+」按钮，点击展开九宫格浮层，提供以下 10 个功能选项：

| 选项 | 图标 | 功能 |
|------|------|------|
| 照片 | 🖼 | 发送照片 |
| 拍摄 | 📷 | 拍摄并发送 |
| 位置 | 📍 | 发送位置 |
| 语音输入 | 🎤 | 语音消息 |
| 收藏 | 📦 | 发送收藏内容 |
| 个人名片 | 👤 | 分享个人名片 |
| 文件 | 📁 | 发送文件 |
| 音乐 | 🎵 | 分享音乐 |
| **发起续写** | 📖 | 发起 AIGC 故事续写（见 4.4.4） |
| **虚拟场景** | 🎭 | 进入虚拟场景选择（见 4.4.5） |

### 4.4.4 AIGC 续写卡片气泡（V3.7 新增）

**设计理念：** 将 AIGC 故事续写的全流程嵌入聊天消息流中，以卡片式气泡呈现，无需跳离聊天场景。

**入口（两个）：**
1. 聊天页「+」→「发起续写」
2. 关系详情页「续写下一章」→ 自动跳转聊天页并发起

**卡片状态流转：**

| 状态 | 卡片样式 | 文案 | 交互 |
|------|---------|------|------|
| **pending**（待确认） | 紫色邀请卡 | 「📖 续写邀请 · {发起者} 邀请你一起续写故事」「双方同意后，240 分钟后自动生成故事并归入关系页」 | YES → 进入 countdown；NO → 进入 rejected |
| **countdown**（续写中） | 进度卡 | 「⏳ 续写中 · 240 分钟」「点击此处完成（模拟）」 | 点击 → 进入 done |
| **done**（已完成） | 完成卡 | 「✓ 故事已生成 · 已归入关系页」 | 点击「去关系页看看」→ 跳转关系详情 |
| **rejected**（已拒绝） | 灰色卡 | 「续写邀请 · 已拒绝」 | 无操作 |

**归档机制：** 故事生成后自动归入 Zone 2 关系详情的故事簿，作为新的一章。

### 4.4.5 虚拟场景选择（V3.7 新增）

**设计理念：** AI 创设虚拟聊天场景（类似跑团），为双方提供沉浸式聊天话题和情境。

**入口：** 聊天页「+」→「虚拟场景」

**场景列表：**
- 每批展示 3 个场景，共 6 个预设场景

| 场景 | 简介 |
|------|------|
| 咖啡厅 | 午后两杯拿铁，隔着窗看陌生人来来往往 |
| 会议室 | 两人被困在同一间会议室里，议题只有彼此 |
| 海上游艇 | 星空之下，海浪之上，无人打扰的甲板 |
| 深夜便利店 | 凌晨三点的便利店，你们同时伸手拿了最后一瓶水 |
| 屋顶天台 | 城市灯火尽收眼底，风刚好适合说秘密 |
| 旧书店 | 你从书架背后看到对方正好翻到同一页 |

**次数限制（两类独立计数）：**

| 限制 | 每日上限 | 说明 |
|------|---------|------|
| 换一批场景 | 3 次 | 每次随机切换为不同批次场景，用完显示「今日已用完」 |
| 进入剧本 | 3 次 | 每次进入一个场景消耗 1 次，用完后场景卡片禁用 |

### 4.4.6 虚拟场景房间（V3.7 新增）

**场景房间结构：**
- **顶部信息区：** 场景名称 + 剧本简介（如「午后两杯拿铁，隔着窗看陌生人来来往往」）
- **AI 提示语：** 「在此与对方自由聊天，AI 会适时插入话题提示。本场可随时结束。」
- **消息区域：** 标准聊天气泡，支持自由文本聊天
- **底部输入栏：** 文本输入 + 发送按钮
- **结束场景按钮：** 用户主动结束当前场景

### 4.4.7 AIGC 场景纪念（V3.7 新增）

**触发时机：** 用户点击「结束场景」后立即弹出。

**纪念卡内容：**
- 标题：基于场景生成（如「咖啡厅 · 今夜对话」）
- 文案摘要：AI 根据双方对话生成的图文故事（如「窗外的雨声和两杯拿铁，把这段对话留在了今晚。AI 把这一刻写成了你们共同的故事。」）
- 缩略图：场景标签占位

**操作：**
- 「去关系页」→ 跳转 Zone 2 关系详情查看归档故事
- 「返回聊天」→ 关闭纪念卡，回到聊天页

## 4.5 我的（Zone 4）

**目标：** 用户个人空间 + AI 分身管理中心。

### 4.5.1 我的主页

- **顶部 AI 分身区（可下拉展开）：**
  - 卡通数字人 SVG 头像（随展开变大）
  - 默认 1/3 屏，展开至 3/4 屏
  - 点击进入 AI 分身调教中心
- **个人资料区：** 编辑资料 / 钱包 / 昵称 / 简介 / 关注粉丝数 / 玛薯号
- **动态区：** 发布 / 赞过 Tab，空态占位

### 4.5.2 AI 分身调教中心

四个入口，集中管理所有 AI 行为：

| 入口 | 功能 |
|------|------|
| 每日战报 | 查看离线期间分身活动档案 |
| 与分身对齐 | 对话或选择类测试校准分身偏好 |
| 离线挂机指令 | 设置下线后分身的活动方向 |
| 让分身发内容 | 选场景配图，用户确认后发布到广场 |

### 4.5.3 分身发布内容流程（V3.7 更新）

**入口**：「我的主页」动态区「＋ 发点什么」按钮。

**V3.7 变更：** 无论选择哪种路径，均**统一先进入场景选择页**，不再直接跳转确认发布。

**场景选择页功能：**
1. 顶部「让 AI 分身帮我决定」快捷入口（AI 自动随机选场景 + 生成文案）
2. 手动场景列表（每批展示 3 条，共 6 个预设）
3. **换一批场景**：每日限 3 次，每次随机切换到不同批次场景

**确认发布页（所有路径共用）：**
- AI 路径显示「AI 已帮你生成内容」提示标签
- 配文文本框可完全重写
- 「确认发布」后内容出现在广场信息流

支持「引用故事」功能：从关系故事簿选取一段 AIGC 故事，作为嵌入卡片附带在动态中。

# 五、设计语言

## 5.1 视觉风格

- **主色**：品牌紫 #7c5cff
- **辅色**：清醒青绿 #3ecfb4（命定之人 / 消息）、暖黄 #ffba08（装饰 / 战报）
- **背景**：淡紫白 #f4f6ff，卡片纯白
- **字体**：PingFang SC / Segoe UI，正文 14-15px，标题 18-22px
- **圆角**：主卡片 18px，次卡片 12-14px
- **阴影**：柔和多层投影（紫色调）

## 5.2 核心 UI 特征

- 广场：小红书双竖排 Masonry 布局
- 故事卡：2×2 网格，彩色渐变配图占位
- 头像：彩色圆形，无边框
- 卡通数字人：SVG 手绘风格，黄色脸 + 紫色天线 + 腮红
- 聊天「+」菜单：九宫格浮层，图标 + 文字标签
- 续写卡片：紫色调卡片气泡，内嵌 YES/NO 操作按钮
- 虚拟场景卡：圆角场景卡片，底部剩余次数标签

## 5.3 交互原则

- 命定之人有倒计时，制造紧迫感（消息列表 + 揭面页双重提示）
- 故事卡支持转发和一键发广场
- 分身发内容必须经过用户最终确认
- 下拉展开 AI 区域，触感自然
- 续写卡片在聊天流中原地变形，不跳离对话场景
- 虚拟场景有明确的次数限制，制造稀缺感
- 场景结束时生成 AIGC 纪念，给双方留下仪式感闭环
