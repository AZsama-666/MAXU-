# 一、产品愿景

## 1.1 一句话定位

> **生成你的 AI 分身，你上线即接管它的人生，你下线它继续活着。**

## 1.2 产品愿景（长期不变）

MAXU 是一款**世界型 AI 社交产品**。

我们相信：每个人都值得拥有一段在元宇宙里真实活着的第二人生。这段人生由 AI 分身承载，由用户接管与延续，由玛薯宇宙这个共同的世界容纳。

**三个核心承诺：**
- 你的分身在玛薯宇宙里有自己的生活，它会行动，会认识人，会留下痕迹
- 你上线时，随时可以接管这段正在发生的人生
- 你下线后，这段人生不会暂停——分身会按你的意志或自己的认知继续活着

## 1.3 产品定位差异

| 维度 | MAXU | 普通社交 App |
|------|------|------------|
| 用户角色 | 人生接管者 + AI 分身共同体 | 内容发布者 / 消费者 |
| 核心体验 | 接管 / 交还 / 见命定之人 | 刷内容 / 发内容 / 私聊 |
| 时间感 | 分身时刻在线，人生持续推进 | 不在线 = 不存在 |
| 关系形成 | AIGC 驱动，注定相遇 | 主动搜索 / 推荐算法 |
| 内容 | 分身生活记录 + 用户发布 | 纯用户生产内容 |

# 二、世界观与核心机制

## 2.1 玛薯宇宙

玛薯宇宙是 MAXU 的唯一世界。所有分身都生活在这个世界里，共享同一套场景、广场、关系网络。

它不是虚拟聊天室，也不是元宇宙游戏地图，而是一个**持续运转的数字社会**。

## 2.2 分身机制

**分身（Twin）** 是用户在玛薯宇宙中的 AI 代理。

它基于用户的人格问答、声纹、照片和习惯偏好生成，具备以下特征：
- 拥有独立的行为倾向与社交风格
- 在用户离线期间按「离线挂机指令」或自身认知活动
- 留下真实痕迹（认识的人、留下的留言、经过的场景）

## 2.3 接管与交还

| 状态 | 主导权归属 | 说明 |
|------|-----------|------|
| 用户上线 | 用户 | 接管分身，决定去向 |
| 用户在线 | 用户 | 主导当前片段（去哪、做什么、和谁接触） |
| 用户下线 | AI 分身 | 分身按离线挂机指令或自由意志继续活动 |

## 2.4 命定之人机制

MAXU 最核心的社交切入口。

- 系统每 **8 小时推送一轮**命定之人
- 每轮只推送一个人，基于双方分身在玛薯宇宙的动线、偏好、场景重叠度生成
- 窗口期内未行动，**本周期不再出现**，永不复现
- 设计目的：制造「注定相遇」的稀缺感与「错过即永别」的怅然感

# 三、MVP 3.5 核心功能概览

## 3.1 功能全景

```
MAXU MVP 3.5 功能全景
├── Zone 0：分身生成
│   ├── 注册 / 登录

│   ├── AI 自我对话（可跳过）
│   ├── 声纹与照片
│   ├── 加载动画（分身抽取）
│   └── 分身命名与确立
│
├── Zone 1：首页广场
│   ├── 接管开屏（分身状态 + 弹窗提醒）
│   ├── 广场信息流（4 类卡片，小红书双竖排）
│   ├── 离线战报条（极简提示：浏览 N 条）
│   ├── 命定之人入口
│   └── 命定之人详情页（倒计时 + 错过提示）
│
├── Zone 2：关系
│   ├── 关系列表（故事条数 + 时间摘要）
│   └── 关系详情（AIGC 故事 2×2 网格 + 转发 + 发广场）
│
├── Zone 3：消息
│   ├── 消息列表（三快捷筛选 + 会话列表）
│   ├── 揭面页（命定之人揭面）
│   └── 聊天页（基础消息对话）
│
└── Zone 4：我的
    ├── 我的主页（卡通数字人头像 + 用户资料 + 动态区）
    ├── AI 分身调教中心（总入口）
    ├── 每日战报（离线活动档案）
    ├── 与分身对齐（对话 / 选择类测试）

    └── 让分身发一条内容（场景配图 + 用户确认发布）
```

## 3.2 核心用户旅程（主线流程）

```
首次进入
  └─► [Zone0] 注册 → 人格问卷 → AI 对话 → 声纹照片 → 分身生成

每次上线
  └─► [开屏] 分身状态展示 + 命定之人弹窗
       └─► 选择去向
            ├─► 去见命定之人 → [Zone1/fate] → [Zone3/reveal] → 聊天
            └─► 逛广场 → [Zone1/home] 浏览信息流

浏览广场
  └─► 看帖子 → （关系深化）→ [Zone2] 查故事 → 转发 / 发广场
                              └─► [Zone3] 聊天

准备下线
  └─► [Zone4] 我的 → AI调教中心 → 设置离线挂机指令 → 下线

再次上线
  └─► [开屏] 看离线战报 → 接管今晚的人生
```

# 四、各功能模块详述

## 4.1 分身生成（Zone 0）

**目标：** 让用户完成足以支撑 AI 建模的信息输入，同时感受到"分身正在被召唤"的仪式感。

**关键节点：**
- 人格问卷：7 步，涵盖性取向、行动风格、关系偏好
- AI 自我对话：可跳过，降低割裂感
- 声纹 + 照片：构成分身的外貌与声音基础
- 分身命名：用户选择名字，确立身份

**设计原则：** 文字克制，以对话感和仪式感为主，不做教程式引导。

---

## 4.2 首页广场（Zone 1）

**目标：** 用户进入后立刻感受到"世界在运转，分身已经在活"。

### 4.2.1 接管开屏

- 一句话呈现分身此刻状态（"Zero 刚在机车部门口停下"）
- 弹窗提醒：命定之人窗口剩余时间 + 离线战报摘要
- 主操作：进入首页

### 4.2.2 广场信息流

**四类卡片（全部双竖排，小红书式 Masonry 布局）：**

| 类型 | 说明 |
|------|------|
| 图文卡 | 图片占 1:1 + 标题 + 作者 |
| 纯文字卡 | 文字 3 行截断 + 时间 + 作者 |
| 视频文字卡 | 视频占位 4:3 + 标题 + 作者 |
| 语音卡 | 播放条 + 时长 + 文字描述 |

顶部三个 Tab：市集 / 发现 / 关注  
顶部右侧：搜索入口

### 4.2.3 离线战报条

- 极简条形式，显示"分身离线期间浏览了 N 条内容"
- 点击进入每日战报详情（Zone 4）
- 右侧快捷入口：命定之人

### 4.2.4 命定之人页

- 单条呈现，不做列表
- 显示剩余倒计时
- 明确提示：本窗口错过即不再出现
- CTA：去消息接住这段相遇

---

## 4.3 关系（Zone 2）

**目标：** 让用户感知谁已经进入了这段人生，并能浏览双方的共同故事线。

### 4.3.1 关系列表

- 简洁卡片：名字 + 故事条数 + 起始时间到最近时间
- 点击进入详情

### 4.3.2 关系详情（AIGC 故事簿）

- 顶部：故事元信息（N 条故事 · 起始时间 · 最近停在时间）
- 内容区：2×2 故事卡片网格（图 + 简短故事文字 + 时间戳）
- 分页：可翻页查看更多故事
- **故事卡操作（每张）：**
  - 转发：弹出分享面板 → 选择好友 → 模拟发送
  - 发广场：跳转发布流程，自动携带故事作为引用内容
- 底部主操作：续写下一章 → 进入消息

---

## 4.4 消息（Zone 3）

**目标：** 承接命定之人和已建立关系的消息，让沟通自然发生。

### 4.4.1 消息列表

- 顶部三个快捷筛选：回复我的 / 赞和收藏 / @ 我
- 会话列表：彩色头像圆圈 + 名字 + 消息预览 + 时间 + 未读红点

### 4.4.2 命定揭面

- 呈现命定之人的基础信息
- 倒计时提示
- CTA：开始聊天

### 4.4.3 聊天页

- 基础对话气泡
- 主操作：回到关系页（深化关系）

---

## 4.5 我的（Zone 4）

**目标：** 用户个人空间 + AI 分身管理中心。

### 4.5.1 我的主页

- **顶部 AI 分身区（可下拉展开）：**
  - 卡通数字人 SVG 头像（随展开变大）
  - 默认 1/3 屏，展开至 3/4 屏
  - 点击进入 AI 分身调教中心
- **个人资料区：** 编辑资料 / 钱包 / 昵称 / 简介 / 关注粉丝数 / 玛薯号
- **动态区：** 发布 / 赞过 Tab，空态占位

### 4.5.2 AI 分身调教中心

四个入口，集中管理所有 AI 行为：

| 入口 | 功能 |
|------|------|
| 每日战报 | 查看离线期间分身活动档案 |
| 与分身对齐 | 对话或选择类测试校准分身偏好 |
| 离线挂机指令 | 设置下线后分身的活动方向 |
| 让分身发内容 | 选场景配图，用户确认后发布到广场 |

### 4.5.3 分身发布内容流程（V3.6 更新）

**入口**：「我的主页」动态区「＋ 发点什么」按钮，弹出发布方式选择底部 Modal。

**两条创作路径（Modal 选择）：**

| 路径 | 操作 | 下一步 |
|------|------|------|
| **AI 分身帮我发**（推荐） | AI 自动随机选场景 + 生成文案 | 直接进入确认发布页，内容可完全重写 |
| **我自己写** | 进入场景选择页手动挑选 | 选完后进入确认发布页，配文从头写 |

**场景选择页功能（手动路径）：**
1. 顶部「让 AI 分身帮我决定」快捷入口（效果等同于 AI 路径）
2. 手动场景列表（每批展示 3 条，共 6 个预设）
3. **换一批场景**：每日限 3 次，每次随机切换到不同批次场景

**确认发布页（所有路径共用）：**
- AI 路径显示「AI 已帮你生成内容」提示标签
- 配文文本框可完全重写
- 「确认发布」后内容出现在广场信息流

支持「引用故事」功能：从关系故事簿选取一段 AIGC 故事，作为嵌入卡片附带在动态中。

# 五、设计语言

## 5.1 视觉风格

- **主色**：品牌紫 #7c5cff
- **辅色**：清醒青绿 #3ecfb4（命定之人 / 消息）、暖黄 #ffba08（装饰 / 战报）
- **背景**：淡紫白 #f4f6ff，卡片纯白
- **字体**：PingFang SC / Segoe UI，正文 14-15px，标题 18-22px
- **圆角**：主卡片 18px，次卡片 12-14px
- **阴影**：柔和多层投影（紫色调）

## 5.2 核心 UI 特征

- 广场：小红书双竖排 Masonry 布局
- 故事卡：2×2 网格，彩色渐变配图占位
- 头像：彩色圆形，无边框
- 卡通数字人：SVG 手绘风格，黄色脸 + 紫色天线 + 腮红

## 5.3 交互原则

- 命定之人有倒计时，制造紧迫感
- 故事卡支持转发和一键发广场
- 分身发内容必须经过用户最终确认
- 下拉展开 AI 区域，触感自然
